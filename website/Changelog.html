Changes since 0.9.4

The build 0.9.4.1312 fixes many problems in 0.9.4.1311. The changes include:
  • Added support for CSI[0i, which prints out the screen.
  • Fixed a memory leak in PseudoTerminal.
  • Added support for opening .command, .tool, .sh, .zsh, .csh, .pl files as shell scripts.
  • Now press TAB in the Execute field will open a new session with the command.
  • Fixed a crash when saving/restoring buffers.
  • More fixes for resizing behaviors.
  • Fixed opening file behavior (#1629686).
  • Clear screen now leaves the last line (which can wrap to several lines) intact. (#1575193)
  • Fixed a bug of cmd-w closing a terminal tab when preference panel is in the front. (#1618969)
  • Fixed hanging-on-printing bug. (#1630780, #1598353)
Warning: The build 0.9.4.1311 is one of those "risky" updates. You shouldn't rely your critical work on it. The changes include:
  • Improved wrapping behavior. Resizing window now re-wraps long lines.
  • Generally improved resizing behavior.
  • Faster response if you use dynamic titling a lot.
  • Reduced CPU usage.
The build 0.9.4.1306 includes the following changes:
  • Improved memory usage for "unlimited" scrollback situation. An upper limit is set to avoid eating up all system memory.
The build 0.9.4.1303 includes the following changes:
  • Added support for xterm's CSI S (SU), CSI T (SD) commands. Nano now works correctly when term type is xterm.
  • Added support for unlimited scrollback. Set scroll back lines to -1 or any number larger than 1,000,000.
  • Added an option to enable/disable the border at the bottom of windows.
  • Fixed initial window size bug (#1623786).
  • Fixed the bottom border, so it no longer overlaps with the resizing triangle.
The build 0.9.4.1221.1 includes the following changes:
  • Added a window border when there's no tab bar at the bottom of screen.
  • Fixed a selection location bug (#1608727).
  • Fixed a scrolling bug (#1619629).
  • Fixed a bug that disables dragging of text.
  • Updated the "term type" list. Removed xterm that doesn't work for some keys. Now rxvt is used as default.
  • Fixed a reverse bold color bug.
  • Fixed a few bug that causes crashes.
The build 0.9.4.1217 includes the following changes:
  • Fixed the tabbar-disappearing bug (#1613110).
  • Fixed a bold color bug.
  • Fixed transparency bug #1609022.
  • Set echok when launching new sessions (#1472673).
  • Changed the default for "use default title with dynamic title" to NO.
  • Changed the wording for the "dynamic title" option.
The build 0.9.4.1208 includes the following changes:
  • Added support for xterm's 256-color mode.
  • Added an option in the terminal profile to allow default session name not to be overwritten by dynamic tab name changes.
  • Double click on a parenthesis now selects everything between the pair of parenthesis.
  • Find panel now will set the string system-wide.
  • Fixed a wrapping porblem under ANSI terminals.
  • Incorporated the data processing thread into the data reading thread.
  • Fixed bug that loses changes to 'Characters considered parts of word' text field.
  • Fixed the "show cursor" behavior in RESET.
  • Fixed tab order bug.
  • An attempt to fix the closing window confirmation panel. Please test.
Release 0.9.4 12/4/06
This version further improves on stability. Further more, we changed the font rendering to make it more in line with other Cocoa applications. Lastly, we have an updated Help that reflects the latest changes in the GUI. We encourage all users who use previous versions to upgrade. New changes include:
  • New font rendering code.
  • Updated Help documentation.
  • Improved support for the wide glyphs, especially for the eastern asian languages.
  • Added a Google command in the context menu to launch a search session with selected text.
  • Improved speed.
  • Enhanced stability.
  • Other bug fixes and GUI enhancements as usual.
